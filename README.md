# Initial PC Setup (Мастер первоначальной настройки)

**Версия 2.0 (стабильная, с расширенными возможностями)**

## О проекте

PowerShell-скрипт с графическим интерфейсом WPF, предназначенный для быстрой первоначальной настройки нового компьютера в корпоративной среде.  
Скрипт автоматизирует рутинные операции:

- Резервное копирование текущих сетевых настроек
- Установка статического IP-адреса, маски, шлюза и DNS-серверов
- Переименование компьютера
- Ввод в домен Active Directory с возможностью указания OU
- Настройка системного прокси (опционально)
- Ведение лога всех действий

Всё это выполняется через удобное графическое окно, полностью настраиваемое в Visual Studio (WPF).

---

## Новые возможности версии 2.0

- ✅ **Автоматический запрос прав администратора (UAC)** – при запуске скрипт сам определяет, есть ли права администратора, и если их нет, перезапускается с повышенными правами через UAC-запрос. Больше не нужно вручную запускать PowerShell от имени администратора!
- ✅ **Валидация всех вводимых полей** – IP, маска (префикс), имя компьютера, DNS – проверяются перед выполнением, с выводом понятных сообщений об ошибках.
- ✅ **Выбор сетевого адаптера** – можно выбрать любой активный физический адаптер из выпадающего списка (кнопка «Обновить» для пересканирования).
- ✅ **Автоматический поиск свободного IP** – кнопка «Найти свободный» сканирует подсеть (поддерживаются /8, /16, /24) и предлагает первый непингуемый адрес.
- ✅ **Указание OU (подразделения)** – при вводе в домен можно указать конкретное подразделение (или оставить пустым для контейнера по умолчанию).
- ✅ **Настройка прокси-сервера** – чекбокс «Настроить системный прокси» позволяет задать адрес и порт для прокси (применяется и для WinHTTP).
- ✅ **Проверка доступности контроллера домена** – перед вводом в домен выполняется DNS-запрос и ping; при недоступности выводится предупреждение.
- ✅ **Прогресс-бар** – отображает ход выполнения операций (настройка IP, прокси, переименование, ввод в домен).
- ✅ **Полная совместимость с PowerShell 5.1** – скрипт работает на всех поддерживаемых версиях Windows без необходимости установки дополнительных модулей.
- ✅ **Улучшенное логирование** – теперь в лог включается информация о выбранном адаптере, OU и настройках прокси.

## Системные требования

- Windows 10/11 (или Windows Server с GUI)
- PowerShell 5.1 (встроен в Windows) или новее
- Учётные данные с правами на добавление ПК в домен
- Доступность контроллера домена по сети (DNS должен быть настроен)

> **Примечание:** права администратора запрашиваются автоматически при запуске, поэтому не нужно запускать PowerShell от имени администратора вручную.

## Инструкция по использованию

1. Скачайте файл `InitialSetup.ps1`.
2. Запустите скрипт любым способом:
   - **Двойным кликом** по файлу (если .ps1 ассоциирован с PowerShell).
   - Через PowerShell: `.\InitialSetup.ps1`.
3. При первом запуске появится запрос UAC на повышение прав – нажмите **Да**.
4. В появившемся окне:
   - Выберите сетевой адаптер из списка (при необходимости нажмите «Обновить»).
   - Введите новый IP-адрес (или нажмите «Найти свободный» для автоматического подбора).
   - Укажите маску (префикс), шлюз и DNS-серверы.
   - Введите новое имя компьютера, домен и (опционально) подразделение OU.
   - При необходимости включите настройку прокси и укажите адрес с портом.
5. Нажмите **OK**.
6. При появлении окна введите учётные данные администратора домена (DOMAIN\Admin или admin@domain.local).
7. Дождитесь завершения операций – прогресс-бар покажет этапы.
8. После завершения будет предложено перезагрузить компьютер (рекомендуется).
